109 /*
110  *-- This contains the options used to control the solve process.
111  *
112  * Fact (fact_t)
113  * Specifies whether or not the factored form of the matrix
114  * A is supplied on entry, and if not, how the matrix A should
115  * be factorizaed.
116  * = DOFACT: The matrix A will be factorized from scratch, and the
117  * factors will be stored in L and U.
118  * = SamePattern: The matrix A will be factorized assuming
119  * that a factorization of a matrix with the same sparsity
120  * pattern was performed prior to this one. Therefore, this
121  * factorization will reuse column permutation vector
122  * ScalePermstruct->perm_c and the column elimination tree
123  * LUstruct->etree.
124  * = SamePattern_SameRowPerm: The matrix A will be factorized
125  * assuming that a factorization of a matrix with the same
126  * sparsity pattern and similar numerical values was performed
127  * prior to this one. Therefore, this factorization will reuse
128  * both row and column scaling factors R and C, both row and
129  * column permutation vectors perm_r and perm_c, and the
130  * data structure set up from the previous symbolic factorization.
131  * = FACTORED: On entry, L, U, perm_r and perm_c contain the
132  * factored form of A. If DiagScale is not NOEQUIL, the matrix
133  * A has been equilibrated with scaling factors R and C.
134  *
135  * Equil (yes_no_t)
136  * Specifies whether to equilibrate the system (scale A's row and
137  * columns to have unit norm).
138  *
139  * ColPerm (colperm_t)
140  * Specifies what type of column permutation to use to reduce fill.
141  * = NATURAL: use the natural ordering
142  * = MMD_ATA: use minimum degree ordering on structure of A'*A
143  * = MMD_AT_PLUS_A: use minimum degree ordering on structure of A'+A
144  * = COLAMD: use approximate minimum degree column ordering
145  * = MY_PERMC: use the ordering specified in ScalePermstruct->perm_c[]
146  *
147  * Trans (trans_t)
148  * Specifies the form of the system of equations:
149  * = NOTRANS: A * X = B (No transpose)
150  * = TRANS: A**T * X = B (Transpose)
151  * = CONJ: A**H * X = B (Transpose)
152  *
153  * IterRefine (IterRefine_t)
154  * Specifies whether to perform iterative refinement.
155  * = NO: no iterative refinement
156  * = WorkingPrec: perform iterative refinement in working precision
157  * = ExtraPrec: perform iterative refinement in extra precision
158  *
159  * PrintStat (yes_no_t)
160  * Specifies whether to print the solver's statistics.
161  *
162  * DiagPivotThresh (double, in [0.0, 1.0]) (only for sequential SuperLU)
163  * Specifies the threshold used for a diagonal entry to be an
164  * acceptable pivot.
165  *
166  * PivotGrowth (yes_no_t)
167  * Specifies whether to compute the reciprocal pivot growth.
168  *
169  * ConditionNumber (ues_no_t)
170  * Specifies whether to compute the reciprocal condition number.
171  *
172  * RowPerm (rowperm_t) (only for SuperLU_DIST)
173  * Specifies whether to permute rows of the original matrix.
174  * = NO: not to permute the rows
175  * = LargeDiag: make the diagonal large relative to the off-diagonal
176  * = MY_PERMR: use the permutation given in ScalePermstruct->perm_r[]
177  *
178  * ReplaceTinyPivot (yes_no_t) (only for SuperLU_DIST)
179  * Specifies whether to replace the tiny diagonals by
180  * sqrt(epsilon)*||A|| during LU factorization.
181  *
182  * SolveInitialized (yes_no_t) (only for SuperLU_DIST)
183  * Specifies whether the initialization has been performed to the
184  * triangular solve.
185  *
186  * RefineInitialized (yes_no_t) (only for SuperLU_DIST)
187  * Specifies whether the initialization has been performed to the
188  * sparse matrix-vector multiplication routine needed in iterative
189  * refinement.
