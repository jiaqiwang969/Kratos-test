20 /*
21  very simple dynamic array class which is used in `higher level'
22  bindings functions for pivot and work arrays
23 
24  Namely, there are (at least) two versions of all bindings functions
25  where called LAPACK function expects work and/or pivot array, e.g.
26 
27  `lower' level (user should provide work and pivot arrays):
28  int sysv (SymmA& a, IVec& i, MatrB& b, Work& w);
29 
30  `higher' level (with `internal' work and pivot arrays):
31  int sysv (SymmA& a, MatrB& b);
32 
33  Probably you ask why I didn't use std::vector. There are two reasons.
34  First is efficiency -- std::vector's constructor initialises vector
35  elements. Second is consistency. LAPACK functions use `info' parameter
36  as an error indicator. On the other hand, std::vector's allocator can
37  throw an exception if memory allocation fails. detail::array's
38  constructor uses `new (nothrow)' which returns 0 if allocation fails.
39  So I can check whether array::storage == 0 and return appropriate error
40  in `info'.
41  */
